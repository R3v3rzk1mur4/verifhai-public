## Module Purpose

Enable users and IT teams to cooperate effectively with AI-driven endpoint security while building a positive security culture. Covers EDR/XDR awareness, threat recognition, phishing defense, and privacy transparency for AI monitoring.

## Level 1: CRAWL - Endpoint Security Awareness

### Learning Objectives

After completing L1, learners will be able to:

1. Understand how AI protects their devices from threats
2. Recognize common threats (phishing, malware, social engineering)
3. Respond appropriately to AI security alerts and actions
4. Know their privacy protections regarding AI monitoring

---

### 1.1 How AI Protects Your Devices

**AI Endpoint Security:**

Your organization uses AI-powered security tools to protect devices from threats:

| Protection | What AI Does | What You See |
|------------|--------------|--------------|
| **Malware Detection** | Identifies malicious software by behavior | Blocked malware notification |
| **Phishing Protection** | Detects suspicious emails and links | Warning when clicking suspicious links |
| **Ransomware Prevention** | Stops encryption attacks | File protection alerts |
| **Behavioral Analysis** | Detects unusual activity patterns | Occasional verification requests |
| **Automatic Updates** | Keeps software patched | Update notifications |

**Why AI is Used:**

- **Speed**: AI detects threats in milliseconds
- **Scale**: Monitors thousands of devices simultaneously
- **Patterns**: Recognizes threat patterns humans might miss
- **24/7**: Protects continuously, even outside business hours

---

### 1.2 Recognizing Threats

**Phishing - The #1 Threat:**

Phishing emails try to trick you into revealing credentials or clicking malicious links.

**Red Flags:**

| Red Flag | Example |
|----------|---------|
| **Urgency** | "Act now or your account will be closed!" |
| **Suspicious Sender** | Support@amaz0n-security.com (not real Amazon) |
| **Generic Greeting** | "Dear Customer" instead of your name |
| **Grammar/Spelling** | Obvious errors in professional communications |
| **Mismatched Links** | Hover shows different URL than displayed |
| **Unexpected Attachments** | Invoice you didn't expect |
| **Requests for Credentials** | "Verify your password by clicking here" |

**What To Do:**

```
┌─────────────────────────────────────────────────────────────┐
│         SUSPECT PHISHING? TAKE THESE STEPS                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. DON'T CLICK                                             │
│     └── Don't click links or open attachments              │
│                                                             │
│  2. VERIFY SEPARATELY                                       │
│     └── If it claims to be from a company, go directly     │
│         to their website (don't use links in email)        │
│                                                             │
│  3. REPORT IT                                               │
│     └── Forward suspicious emails to security team         │
│     └── Use the "Report Phishing" button if available      │
│                                                             │
│  4. DELETE IT                                               │
│     └── After reporting, delete the email                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**Other Threats to Recognize:**

| Threat | How It Works | Protection |
|--------|--------------|------------|
| **Malware** | Software that harms your device | Don't install unauthorized software |
| **Social Engineering** | Manipulation to get information | Verify requests through official channels |
| **USB Attacks** | Malicious USB devices | Don't plug in unknown USB drives |
| **Password Attacks** | Guessing or stealing passwords | Use strong, unique passwords + MFA |

---

### 1.3 Responding to AI Security Alerts

**When AI Takes Action:**

AI security tools may take protective actions on your device:

| AI Action | What It Means | What To Do |
|-----------|---------------|------------|
| **Blocked Application** | AI detected potential threat | If legitimate, contact IT for exception |
| **Quarantined File** | File moved to safe storage | Don't attempt to restore without IT help |
| **Isolated Device** | Device disconnected from network | Contact IT immediately |
| **Blocked Website** | Site flagged as dangerous | Find information elsewhere; report if false positive |
| **Required Update** | Security patch needed | Install update promptly |

**If You Think AI Made a Mistake:**

1. **Don't try to disable security** - This makes you vulnerable
2. **Document what happened** - Screenshot the alert
3. **Contact IT/Security** - They can review and create exceptions if needed
4. **Be patient** - False positives happen; proper review takes time

---

### 1.4 Privacy and AI Monitoring

**What AI Monitors (and What It Doesn't):**

| AI Monitors | AI Does NOT Monitor |
|-------------|---------------------|
| Security threats and malware | Personal file content |
| Suspicious network connections | Your private communications |
| Unusual application behavior | Everything you type |
| Potential data exfiltration | Your location constantly |

**Your Privacy Protections:**

- AI focuses on security threats, not employee surveillance
- Data is used for security purposes only
- Access to AI data is restricted and audited
- You can ask what data is collected about you

**Transparency:**

Your organization commits to:
- Clear communication about what security tools monitor
- Security purpose only - not performance monitoring
- Regular privacy reviews of AI security tools
- Your right to ask questions about monitoring

---

### 1.5 Your Role in Security

**Security is Everyone's Responsibility:**

| Your Action | Security Impact |
|-------------|-----------------|
| Report suspicious emails | Helps protect colleagues |
| Keep software updated | Closes security vulnerabilities |
| Use strong passwords + MFA | Prevents account compromise |
| Lock your device | Protects data when away |
| Ask questions | Helps identify new threats |

**The Human Firewall:**

> AI catches most threats, but you are the last line of defense. Your awareness and actions matter.

### 2.1 Role-Based Security Training

**For High-Risk Roles (Finance, HR, Executives):**

These roles are targeted more frequently by attackers:

| Threat | Why This Role | Enhanced Protection |
|--------|---------------|---------------------|
| **Business Email Compromise** | Finance can authorize payments | Verify payment requests out-of-band |
| **HR Data Theft** | Access to employee PII | Extra vigilance on data requests |
| **Executive Impersonation** | Authority to approve actions | Verify unusual requests |

**Additional Practices:**

```markdown
## Enhanced Security Checklist for High-Risk Roles

### Before Processing Financial Requests
- [ ] Verify request through phone call to known number
- [ ] Confirm with second authorized person
- [ ] Check for urgency pressure (red flag)
- [ ] Verify email sender details carefully

### Before Sharing Sensitive Data
- [ ] Confirm recipient's identity
- [ ] Verify legitimate business need
- [ ] Use secure sharing methods
- [ ] Log/document the sharing

### Daily Practices
- [ ] Review account activity for anomalies
- [ ] Report any suspicious contacts
- [ ] Keep devices locked when away
- [ ] Use approved devices only for sensitive work
```

**For IT Support:**

Supporting users with AI endpoint security:

| User Issue | Support Response |
|------------|------------------|
| "AI blocked my application" | Review legitimacy, submit exception if valid |
| "I clicked a phishing link" | Initiate incident response, reset credentials |
| "My device was quarantined" | Investigate alert, coordinate with security |
| "Is this email legitimate?" | Help verify, encourage reporting |

### 2.2 Phishing Simulations

**Why We Run Simulations:**

- Test and improve awareness
- Identify training needs
- Measure security culture
- Practice safe reporting

**What To Expect:**

- Periodic simulated phishing emails
- Realistic but not designed to trick maliciously
- Educational feedback when you click
- No punishment - learning opportunity

**How To Succeed:**

| Do | Don't |
|----|-------|
| Apply phishing detection skills | Feel embarrassed if you click |
| Report suspicious emails | Try to identify simulations vs. real |
| Learn from feedback | Disable security tools |
| Share learnings with colleagues | Punish yourself or others |

**If You Click:**

1. **Don't panic** - Simulations are for learning
2. **Read the training** - Understand what you missed
3. **Apply learning** - Watch for those indicators next time
4. **Ask questions** - Security team is here to help

Run the Device Health Check from system tray icon

### 2.4 Supporting Security Culture

**Being a Security Ally:**

| Action | Impact |
|--------|--------|
| Share what you learn | Colleagues benefit from your knowledge |
| Encourage reporting | No one should feel bad about asking |
| Model good behavior | Others follow security-conscious leaders |
| Thank security team | Encourages their difficult work |

**Having Security Conversations:**

- **Non-judgmental**: "I almost clicked that too - good catch!"
- **Helpful**: "Here's how I verify suspicious emails..."
- **Encouraging**: "Thanks for reporting - you may have protected others"
- **Curious**: "What made you realize it was suspicious?"

## Level 3: RUN - Security Champions & Culture Leadership

### Learning Objectives

After completing L3, learners will be able to:

1. Serve as a Security Champion in their team
2. Lead security awareness initiatives
3. Contribute to security culture improvement
4. Measure and improve security awareness effectiveness

---

### 3.1 Security Champions Program

**Champion Responsibilities:**

| Responsibility | Activities |
|----------------|------------|
| **Advocacy** | Promote security awareness in team meetings |
| **First Response** | Help colleagues with security questions |
| **Liaison** | Connect team with security team resources |
| **Feedback** | Share team concerns with security program |
| **Recognition** | Celebrate security-conscious behavior |

**Time Commitment:** ~2-4 hours/month

**Champion Skills:**

```markdown
## Security Champion Skills

### Communication
- Explain security concepts in simple terms
- Make security relatable, not scary
- Listen to concerns and feedback
- Present security topics to groups

### Technical Awareness
- Recognize common threats
- Understand AI security tools (user level)
- Know when to escalate to security team
- Use self-service resources effectively

### Leadership
- Model security-conscious behavior
- Encourage reporting without judgment
- Build trust with team members
- Advocate for practical security
```

### 3.2 Security Awareness Initiatives

**Running Effective Campaigns:**

| Campaign Type | Best Practices |
|---------------|----------------|
| **Email Campaign** | Short, visual, actionable; link to resources |
| **Lunch & Learn** | Interactive, real examples, Q&A time |
| **Team Meetings** | 5-minute security moment; rotate topics |
| **Gamification** | Points, badges, competitions; keep it fun |
| **Real-Time Alerts** | When relevant threats emerge; brief and clear |

**Campaign Calendar:**

| Month | Focus | Activities |
|-------|-------|------------|

| July | Physical Security | Desk security check |

### 3.3 Measuring Security Culture

**Key Metrics:**

| Metric | What It Measures | Target |
|--------|------------------|--------|
| **Phishing Click Rate** | Susceptibility to phishing | <5% |
| **Phishing Report Rate** | Proactive reporting | >60% |
| **Training Completion** | Awareness program reach | >95% |
| **MFA Adoption** | Security control adoption | >95% |
| **Security Ticket Trends** | Security-related issues | Decreasing |
| **Culture Survey** | Perception of security | Positive trend |
